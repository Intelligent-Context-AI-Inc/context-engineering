# Contributing to Context Engineering

Thank you for your interest in contributing to the context engineering knowledge repository.

## About This Repository

This repository documents the emerging discipline of context engineering and the conceptual architecture of Enterprise Context Fabric systems. It is maintained by Intelligent Context AI, Inc. as a public educational resource.

## How to Contribute

### Suggesting Improvements

If you have suggestions for improving existing documentation:

1. Open an issue describing the proposed change and its rationale
2. Reference the specific document(s) you believe should be updated
3. Explain how the change improves clarity, accuracy, or completeness

### Proposing New Content

If you believe a concept or topic is missing from the repository:

1. Open an issue describing the proposed addition
2. Explain how it relates to existing concepts in the repository
3. Provide a brief outline of the proposed content

### Submitting Changes

1. Fork the repository
2. Create a branch for your changes
3. Make your changes following the writing guidelines below
4. Submit a pull request with a clear description of the changes

## Writing Guidelines

### Tone and Style

- Use neutral, technical language
- Avoid marketing language, hype, or exaggerated claims
- Write in a declarative, educational style
- Use structured sections with clear headings

### IP Protection

This repository documents conceptual architectures and shared vocabulary. Contributors should:

- Describe capabilities and expected behaviors, not internal algorithms or scoring methods
- Use phrases like "an implementation may provide…" or "systems typically include…"
- Avoid disclosing proprietary implementation details of any specific product
- Clearly distinguish between conceptual architecture, example implementations, and proprietary internals

### Formatting

- Use Markdown for all documentation
- Use tables for structured comparisons
- Use ASCII diagrams for architecture illustrations
- Include cross-references to related documents where appropriate

## Code of Conduct

Contributors are expected to engage respectfully and constructively. This repository is a shared educational resource, and contributions should support that purpose.

## Questions

If you have questions about contributing, please open an issue.
